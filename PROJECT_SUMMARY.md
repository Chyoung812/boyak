# 보약(Boyak) 프로젝트 전체 정리

> 독거 어르신의 3가지 두려움 해결: **약물 부작용** · **병원비 폭탄** · **험한 이동 길**

---

## 1. 지금까지 구현된 것

### 1-1. 약 복용 안전 확인 (약 탭)

| 단계 | 구현 내용 | 상태 |
|------|-----------|------|
| 약봉투 촬영 | 카메라/갤러리 다중 선택, 미리보기 표시 | ✅ |
| OCR 분석 | OpenAI GPT-4o-mini로 약봉투 이미지 → 약 이름 추출 | ✅ |
| 약 이름 정규화 | 로컬 SQLite DB 검색 → 없으면 심평원 easy_drug API fallback | ✅ |
| 후보 선택 UI | 같은 성분 자동 선택, 다수 성분이면 사용자 선택 | ✅ |
| 다른 약 추가 | 감기약/소화제/관절영양제 토글 선택 | ✅ |
| 한약 여부 + 나이 | 입력 후 DUR 분석 트리거 | ✅ |
| DUR 안전확인 | 노인주의/연령금기/병용금기 확인 | ✅ (DB 데이터 필요) |
| 결과 표시 | 위험등급·원인·행동지침 표시, 음성 안내 | ✅ |

**현재 제한**: 로컬 DUR SQLite DB에 데이터가 없음 → 심평원 CSV 파일이 필요함 (아래 섹션 참고)

### 1-2. 병원 길찾기 (길찾기 탭)

| 단계 | 구현 내용 | 상태 |
|------|-----------|------|
| 증상 입력 | 자유 음성 발화 → transcript 표시 → 네/아니오 확인 | ✅ |
| 빠른 선택 | 허리/무릎/어깨/두통 버튼 | ✅ |
| 증상 → 진료과 | 키워드 매핑 (9개 진료과) | ✅ |
| 근처 병원 검색 | TMap POI 검색 (반경 2km, 최대 5개) | ✅ |
| 보행자 경로 | TMap 무장애 노선(searchOption=30) 병렬 조회 | ✅ |
| 평지 우선 정렬 | 계단 수 오름차순 → 거리 오름차순 | ✅ |
| 보행자 맞춤 추천 | 계단 가장 적은 병원에 "보행자 맞춤 추천" 배지 | ✅ |
| 지도 표시 | Leaflet + OpenStreetMap 타일 | ✅ |
| 경로 폴리라인 | TMap 응답에서 LineString 좌표 추출, 파란 선 표시 | ✅ |
| 출발지 선택 | 현재 GPS / 주소 직접 입력 (TMap 지오코딩) | ✅ |
| GPS 실시간 추적 | watchPosition으로 마커 이동, 스텝별 음성 안내 | ✅ |
| 도착 감지 | 목적지 20m 이내 진입 시 "도착" 처리 | ✅ |

## 2. 공공의료 데이터 활용 현황

### 2-1. 현재 사용 중인 공공 API

#### 건강보험심사평가원(심평원) — 핵심

| API 서비스 | 용도 | 호출 시점 | 호출 빈도 |
|-----------|------|----------|----------|
| `DrbEasyDrugInfoService` | 약 이름 검색 (로컬 DB fallback) | OCR 후 약 이름 정규화 시 DB에 없는 약 | 약 이름 1개당 1회, 사용자당 1~5회 |
| `DURPrdlstInfoService03` | DUR 노인주의/연령금기/병용금기 원본 데이터 | 로컬 DB 구축용 (1회성 스크립트) | 로컬 DB 재구축 시에만 |

#### T맵(SK텔레콤)

| API 서비스 | 용도 | 호출 시점 | 호출 빈도 |
|-----------|------|----------|----------|
| `pois/search/around` | 반경 2km 내 병원 POI 검색 | 증상 선택 후 병원 목록 조회 | 사용자 1회당 1번 |
| `routes/pedestrian` | 무장애 보행자 경로 조회 | 병원 후보 최대 5개 병렬 조회 | 사용자 1회당 최대 5번 |
| `pois` (지오코딩) | 주소 텍스트 → 좌표 변환 | 출발지 "주소 변경" 시 | 사용자가 주소 입력할 때만 |

#### 식품안전나라

| API 서비스 | 용도 | 호출 시점 | 호출 빈도 |
|-----------|------|----------|----------|
| `I0760` (건강기능식품 성분) | 영양제 원료 정보 조회 | `/api/supplements/ingredients` 호출 시 | 현재 미사용 (연결은 완료) |

#### OpenAI

| 모델 | 용도 | 호출 시점 |
|------|------|----------|
| `gpt-4o-mini` | 약봉투 이미지 OCR | 사진 업로드 후 분석 버튼 클릭 시 |
| `gpt-4o-mini` | 자연어 → 서비스 라우팅 | `/api/ai/route` 호출 시 |

### 2-2. 아직 연결 안 된 공공 데이터 (필요하면 추가)

| 데이터 | 출처 | 왜 필요한가 |
|--------|------|------------|
| **DUR 의약품 목록 CSV** | 심평원 파일다운로드 | 로컬 SQLite DB를 채워야 약 정규화·DUR 분석이 제대로 작동함. 현재 가장 중요한 미완성 항목 |
| **병용금기약물 CSV** | 한국의약품안전관리원 | DB 구축 시 병용금기 pair table을 채워야 함 |
| **알약 식별 정보** | `MdcinGrnIdntfcInfoService03` | 색·모양·각인으로 약 식별 (어르신이 약 이름 모를 때 유용) |
| **허가정보** | `DrugPrdtPrmsnInfoService07` | 약 매칭 실패 시 보조 정보 제공 |
| **병원 정보** | 건강보험심사평가원 요양기관 현황 | 진료과·운영시간·전화번호 등 실데이터 |

### 2-3. DUR 로컬 DB 구축 방법 (가장 중요한 미완성)

```bash
# 1. 심평원 공공데이터포털에서 CSV 다운로드
#    검색어: "DUR 의약품 목록" → "건강보험심사평가원_의약품안전사용서비스(DUR) 의약품 목록"
#    경로: boyak_mix/backend/data/raw/dur_drug_list/ 에 저장

# 2. 병용금기 CSV 다운로드
#    검색어: "병용금기약물" → "한국의약품안전관리원_병용금기약물"
#    경로: boyak_mix/backend/data/raw/dur_contraindications/ 에 저장

# 3. SQLite DB 빌드 (약 2~5분 소요)
cd boyak_mix/backend
python scripts/build_dur_sqlite.py

# 4. 백엔드 재시작
uvicorn app.main:app --reload --port 8001
```

## 3. 수정/추가가 필요한 것들

### 🔴 높은 우선순위

| 항목 | 문제 | 해결 방법 |
|------|------|----------|
| **DUR DB 데이터 없음** | 약 정규화 후보가 심평원 API fallback에만 의존, DUR 안전분석은 작동 안 함 | 심평원 CSV 파일 다운로드 후 `build_dur_sqlite.py` 실행 |
| **OCR 약 이름 추출 정확도** | 약봉투 각도/조명에 따라 GPT가 이름을 잘못 읽는 경우 있음 | 프롬프트 개선 또는 기준점 마커 활용 |
| **병원비 챗봇 - 실제 AI 미연결** | 현재 키워드 매칭 방식, 예상 못한 질문은 "콜센터 문의" 안내 | OpenAI API 연결하면 자유로운 Q&A 가능 |

### 🟡 중간 우선순위

| 항목 | 문제 | 해결 방법 |
|------|------|----------|
| **GPS 권한 거부 처리** | Safari/iOS에서 GPS 권한 거부 시 서울시청 기본값으로 대체 | 거부 메시지 + 수동 주소 입력 안내 강화 |
| **TMap 키 만료 처리** | API 키 만료 시 조용히 fallback으로 넘어가고 사용자에게 안내 없음 | 에러 메시지 표시 |
| **네비게이션 도착 감지** | 20m 반경 기준인데 GPS 오차로 오탐 가능 | 동적 반경 (도심 50m, 교외 30m) |
| **증상 키워드 미확장** | "종아리", "발바닥", "손", "목" 등 매핑 안 됨 | `_SYMPTOM_DEPT_MAP` 확장 |
| **다른 약 추가 → DUR 연동** | "감기약" 토글 선택해도 DUR 분석에 실제로 반영 안 됨 | page.jsx의 `handleRunMedicineSafetyCheck`에 홈약 포함 |
| **"의사 상담 권장" 버튼** | 버튼만 있고 클릭해도 아무것도 안 함 | 병원 탭으로 이동 연결 |

### 🟢 낮은 우선순위 (개선)

| 항목 | 설명 |
|------|------|
| **알약 사진으로 식별** | 약 이름 모르고 알약 사진만 있을 때 모양/색/각인으로 식별 |
| **즐겨찾는 병원 저장** | 자주 가는 병원 저장, localStorage 또는 간단한 DB |
| **복약 알람** | 약 복용 시간 알림 (브라우저 Notification API) |
| **보행 옵션 선택** | boyak_test처럼 "빠른 길 / 편한 길 / 추천" 3가지 경로 비교 |
| **병원 운영 시간** | 현재 진료 중인지 실시간 표시 (심평원 요양기관 API) |

## 5. 파일 구조 요약

```
boyak_mix/
├── backend/
│   ├── app/
│   │   ├── main.py          ← FastAPI 앱 + 로깅 미들웨어 + 모든 엔드포인트
│   │   ├── config.py        ← 환경변수 (.env 로딩)
│   │   ├── ocr_service.py   ← OpenAI로 약봉투 이미지 → 약 이름
│   │   ├── drug_index_service.py ← 약 이름 정규화 (SQLite + 심평원 API fallback)
│   │   ├── safety_service.py ← DUR 노인주의/금기 확인
│   │   ├── cost_service.py  ← 병원비 예상
│   │   ├── ai_service.py    ← OpenAI 자연어 라우팅
│   │   └── public_data_client.py ← 공공데이터 API 클라이언트
│   ├── data/
│   │   ├── raw/             ← 심평원 CSV 파일 여기 넣어야 함 (현재 비어있음)
│   │   └── processed/
│   │       └── dur_index.sqlite3  ← 로컬 약 DB (현재 테이블만 있고 데이터 없음)
│   ├── logs/                ← API 요청 로그 (자동 생성)
│   ├── scripts/
│   │   ├── build_dur_sqlite.py   ← CSV → SQLite 빌드
│   │   └── validate_endpoints.py ← 엔드포인트 검증 스크립트 (신규)
│   ├── tests/
│   │   ├── test_safety_bags.py
│   │   └── test_drug_index_improvements.py
│   └── .env                 ← API 키 (gitignore)
│
└── frontend/
    ├── app/
    │   ├── page.jsx         ← 메인 페이지 (홈/약/병원/병원비 라우팅)
    │   ├── constants.js     ← 상수 + doctorTips(14개 의약품 팁)
    │   └── components/
    │       ├── MedicineFlowScreen.jsx  ← 약 복용 안전 확인 흐름
    │       ├── HospitalFlowScreen.jsx  ← 증상입력→병원선택→길안내
    │       ├── NavigationMap.jsx       ← Leaflet 지도 + TMap 경로
    │       └── CostEstimateScreen.jsx  ← 병원비 예상 + 챗봇
    └── .env.local           ← 프론트 환경변수 (API URL, TMap 키)
```

## 6. 서버 실행

```bash
# 백엔드 (포트 8001)

# 프론트엔드 (포트 3000)
cd boyak_mix/frontend
npm run dev

# 엔드포인트 검증 (백엔드 실행 중일 때)
cd boyak_mix/backend
python scripts/validate_endpoints.py
```
